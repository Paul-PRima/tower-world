# Changelog

All notable changes to this project are documented here, newest first. Each entry notes who made the change: **(Claude)** for AI-driven edits, **(User)** for manual edits made directly in the Godot editor or elsewhere.

## 2026-07-10

### Added
- Death zone for the manor forest: the exterior floor is only 50x50 but trees scatter out to radius 22, so wandering past its edge dropped into void with nothing to catch you. Added a `DeathZone` Area3D in `manor.tscn` (reusing the existing `teleporter.gd`, same pattern as the boss floor's) that catches anyone who falls and returns them to the manor's front-door landing spot. (Claude)

### Added (game ending)
- Manor forest: added exterior grounds around the manor (`ExteriorFloor` in `manor.tscn`, y=-80) and a 12-tree ring using the same launch-with-E mechanic as the surface forest. Reused `forest.gd`/`tree.gd` rather than forking them â€” added an opt-in `center` offset + `assign_tree_ids` to `forest.gd` and an opt-in `tree_id` to `tree.gd`; when a tree has an id it reports to a new `GameState.collect_manor_tree()`, which fires `all_manor_trees_launched` once all 12 are gone. Regular (non-manor) forest trees are untouched since they never get an id. (Claude)
- Town + ending: once all manor trees are launched, `player.gd` teleports to a new `scenes/town.tscn` (y=-100, four decorative houses reused from the village around a square). Its centerpiece is `scenes/big_tree.tscn`/`big_tree.gd` â€” a scaled-up tree that isn't launched but instead takes a single sword hit (same `take_damage`/`HurtArea` convention as the boss and cobwebs) to trigger `GameState.show_ending()`, which switches to a new `scenes/ending_screen.tscn` ("Thanks for playing!" â€” Space, already bound to the `jump` action, returns to the title screen). (Claude)
- `GameState.save_game()`/`load_save_data()` now also persist `manor_trees_launched`, and `tree.gd`/`entity.gd`-style "already resolved" checks were added so a reloaded save doesn't respawn already-launched manor trees. (Claude)

### Added (this session)
- Title screen (`scenes/title_screen.tscn`/`title_screen.gd`) is now the project's entry point (`run/main_scene`), showing "Joke game.exe" with Continue / New Game / Quit â€” Continue is disabled when no save file exists. New Game calls `GameState.reset_progress()`; Continue calls `GameState.load_save_data()` before switching to `main.tscn`. (Claude)
- Real save/load, file-backed at `user://savegame.json` (JSON). `GameState` now stores `collected_keys` as a per-id dict (was a bare int counter) so partial key progress can be restored correctly, plus `boss_defeated`. `save_game()` writes player position, collected item/key ids, `has_sword`, and boss state; `load_save_data()` reads it back into `GameState` *before* the scene switch (so items/keys/the boss correctly hide/skip themselves in their own `_ready()`), and stashes the player's position/sword for `apply_pending_load()` to apply once `main.tscn`'s new `main.gd` script confirms the player node exists. `item.gd`/`key.gd` now check `GameState` on `_ready()` and hide themselves if already collected; `entity.gd` does the same for `boss_defeated`. Note: this only persists overall progression (position, items, keys, boss, sword) â€” it does not track individual door/tree/cobweb states, which reset per session. (Claude)
- Pause menu now has Save and Back to Title Screen buttons alongside Continue/Quit (`pause_menu.gd`). Save calls `GameState.save_game()` with the current player; Back to Title unpauses (important â€” `get_tree().paused` persists across scene changes, so forgetting this would leave the title screen frozen) and switches to `title_screen.tscn`. `player.gd` gained a `grant_sword()` method (extracted from the item-collection handler) so loading a save with `has_sword=true` can re-grant it the same way. (Claude)

### Added
- Pause menu: pressing Esc now opens a "Paused" overlay (`pause_menu.gd` on a new `PauseMenu` CanvasLayer in `player.tscn`) with Continue and Quit buttons, using `get_tree().paused` + `PROCESS_MODE_ALWAYS` so gameplay actually freezes while it's open. This replaces the old bare "Esc releases the mouse" behavior in `player.gd` (removed, now redundant â€” the pause menu handles mouse mode itself on open/close). (Claude)
- Player death sound slot: added `DeathSound` (`AudioStreamPlayer3D`, empty stream) and `play_death_sound()` to the player (moved off the entity per correction â€” it's the player's death, not the boss's). `entity.gd`'s melee attack now calls it via `has_method("play_death_sound")` right before teleporting the player back to the parkour hub. (Claude)

### Fixed
- `entity.gd`'s `_physics_process` had its `delta` parameter renamed to `_float` during a manual edit, which broke the `velocity.y -= gravity * delta` line's reference and it got deleted along with the rest of the gravity/`is_on_floor` handling â€” leaving the entity floating just above the boss floor with no grounding, which was breaking its chase AI. Restored gravity handling with the parameter properly named `delta`. (Claude)
- Restored the `InputEventMouseMotion` type guard in `player.gd`'s `_unhandled_input`, which had been accidentally deleted along with the Escape-to-release-mouse handling during a manual edit, breaking mouse look entirely. (Claude)
- Crosshair `ColorRect`s (`HBar`/`VBar`) were left at the default `MOUSE_FILTER_STOP`, so sitting at screen center â€” right where the captured cursor sits â€” they silently swallowed every mouse-motion event before `_unhandled_input` saw it, which was the real remaining cause of mouse look not working. Set `mouse_filter = 2` (ignore) on both. (Claude)

### Changed
- Removed the "Well Done" end screen (deleted `hud.gd` and the `WellDoneScreen` UI). Collecting all 4 parkour items now instead grants the player a sword (visible viewmodel on the camera, hidden until earned) that swings on left-click (new `attack` input action) via a `SwordHitbox` overlap check against anything with a `take_damage` method. (Claude)
- The underground's `ReturnPad` (renamed `DescendPad`) no longer sends the player back to the surface â€” it now drops them one floor further down (`target_y` changed from the surface to `-39`) into a new boss arena (`scenes/boss_floor.tscn`). (Claude)

### Added
- Manor house: the maze now has an exit pad (`scenes/maze.tscn`, corner cell) that teleports the player to a new floor (`scenes/manor.tscn`, y=-80) â€” a 2-story, multi-room house built by `house_generator.gd` (hardcoded wall-segment list + a straight staircase, same programmatic-node-building style as the other generators). The front door (`locked_door.gd`/`scenes/locked_door.tscn`) needs 3 keys (`GameState.collected_keys`) before it'll respond to interact; the keys (`key.gd`/`scenes/key.tscn`, with their own pickup sound slot) are hidden in 3 separate rooms across both floors, reachable via an interior staircase. - Death zone under the boss arena and a generic full-position `teleporter.gd` (vs. the Y-only `vertical_teleporter.gd`): falling off the boss floor now drops you into a catcher volume that teleports you back to the arena's entrance instead of into the void; the same script now also powers the maze exit. - Maze: defeating the boss now teleports the player to a new layer one further floor down (`scenes/maze.tscn`, y=-60) â€” a 6x6 perfect maze generated at runtime by `maze_generator.gd` (iterative randomized depth-first/recursive-backtracker carve, same procedural-node-building style as `forest.gd`/`parkour_challenge.gd`), where roughly a third of the doorways are blocked by cobwebs (`cobweb.gd`/`scenes/cobweb.tscn`) instead of open passage. Cobwebs plug into the same sword hit-detection player.gd already uses (a `HurtArea` + `take_damage(amount)`, 1 hit to clear) â€” on destruction they hide and disable collision rather than being freed, so a queued destroy sound isn't cut off. `entity.gd` now teleports the player to the maze's entrance cell instead of just despawning when its health hits 0. (Claude)
- Sword sound slot: added `SwordSwingSound` (`AudioStreamPlayer3D`, empty stream) to the player, played on every swing regardless of whether it connects. (Claude)
- Entity AI: converted `entity.tscn`'s root from a stationary `Node3D` to a `CharacterBody3D` with real collision, and gave `entity.gd` simple chase behavior â€” it finds the player via a new `player` group, moves toward them with gravity/`move_and_slide()` while outside melee range, stops and faces them (`look_at`) once close, and its periodic `AttackArea` hit now also plays a weapon-swing tween. Added a simple mace (cylinder handle + sphere head) as a `Weapon` child so it's no longer unarmed. Added a `BossMusic` slot (`AudioStreamPlayer3D`, autoplay, empty stream, scoped to ~60m so it doesn't bleed into the rest of the map) on `boss_floor.tscn`. (Claude)
- Boss fight: `scenes/entity.tscn`/`scripts/entity.gd` is an enemy with 10 health that dies after 10 sword hits (via its `HurtArea`); it also has its own melee `AttackArea` that, on a timer, teleports the player back to the underground parkour hub if they're caught in range â€” so a single hit ends that attempt. (Claude)
- Underground parkour: shrank the underground hub floor to 44x44 (just enough to safely hold the 4 tower landing spots + return pad) and added 4 procedurally-generated platform courses (`parkour_challenge.gd`, one per tower direction, mirrors the `forest.gd`/`spiral_stairs.gd` procedural-generation pattern) extending out over open void, each ending in a glowing pickup item (`item.gd`/`scenes/item.tscn`). Falling off a platform is caught by a large `fall_reset.gd` volume under the whole layer, which works out which challenge you fell from by X/Z quadrant and teleports you back to that tower's landing spot. `GameState` still tracks collected items and fires `all_items_collected` once all 4 are found, now used to grant the sword instead of showing a screen. - Sprinting: hold Shift (new `sprint` input action) to move at `SPRINT_SPEED` (8.0 vs the normal 5.0) with a faster footstep cadence, playing a separate `SprintFootstepPlayer` sound slot instead of the walking footstep sound. (Claude)
